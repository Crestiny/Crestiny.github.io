¶3. 信息安全基本属性与信息安全层次模型的关系

每个层次有涉及到基本属性中的一些

  • 物理安全：对物理装备的保护
  • 运行安全：对运行过程和运行状态的保护
  • 数据安全：在数据收集、处理、存储、检索、传输、交换、显示、扩散等过程中的保护
  • 内容安全：对信息在网络内流动中的选择性阻断，以保证信息流动的可控能力
  • 信息对抗：对信息熵的真实性的隐藏与保护，或者攻击与分析

¶1.1.4 ISO/OSI安全体系结构

首先解释一下ISO和OSI是什么：

  • OSI=Open System Interconnect，开放系统互连——这是个安全体系结构？
  • ISO=International Organization for Standardization，国际标准化组织——这是个组织

ISO发布了ISO 7498-2标准作为OSI基本参考模型的新补充

ISO 7498-2中描述了：

  • 5种安全服务（安全功能）
  • 8类安全机制+普遍安全机制（支持上面的安全服务的）
  • 5种OSI安全管理方式

¶1. 安全服务

  • 鉴别服务
  • 访问控制服务
  • 数据完整性服务
  • 数据机密性服务
  • 抗抵赖性服务

❓一个疑惑

OSI在记忆中貌似是网络结构，为什么这里叫OSI安全体系呢？

我们来看看wiki上面的OSI模型词条：

开放式系统互联模型（英语：Open System Interconnection Model，缩写：OSI；简称为OSI模型）是一种概念模型，由国际标准化组织提出，一个试图使各种计算机在世界范围内互连为网络的标准框架。定义于ISO/IEC 7498-1。

该模型将通信系统中的数据流划分为七个层，从跨通信介质传输位的物理实现到分布式应用程序数据的最高层表示。每个中间层为其上一层提供功能，其自身功能则由其下一层提供。功能的类别通过标准的通信协议在软件中实现。

开放式系统互联模型的开发始于上世纪70年代后期，用以支持各种计算机联网方法的出现。在上世纪80年代，该模型成为国际标准化组织（ISO）开放系统互连小组的工作产品。

我有个想法，是不是……😳

OSI安全体系结构=OSI参考模型+安全机制+安全服务，后两者是部署在OSI参考模型上的？
